Version 1 2026-07-10

Add custom GA4 events for downloads, fonts, contributor links & views

Instrument user actions on top of the existing GA4 setup (gtag.js, G-KX3WW4Q3NG). A single delegated capture-phase click listener in the shared ANALYTICS block (every page's <head>) fires:

download_svg / download_png — asset SVG/PNG buttons (grid + lightbox) download_pdf — aggregate tear-sheet PDF link get_font — font "get" links (cards + fonts lightbox) contributor_link — outbound links to a contributor's own site (credit cards, font byline, restricted overlay)

Lightbox opens are <div>/card clicks, not anchors, so view_mark and view_font are fired directly from openLb/openFb (on open only).

Triggers hang off existing/added hooks (download attr + extension, and .pdflink / .font-get[data-font] / .contrib-link[data-contrib]) so new UI stays tracked without touching the listener. Document the whole scheme in a new "Analytics (GA4)" section of CLAUDE.md. Regenerated HTML + manifests
